ffmpeg 添加webm支持（添加libvpx库）

记录一下：
因为要转换成webm格式，所以添加了libvpx库
ffmpeg每次添加库后都要重新编译。
首先下载libvpx
刚开始用的yum


yum install libvpx

后来编译ffmpeg时提示版本太低：

must be >=0.9.1

于是去下载源码编译：链接


./configure
make &&make install

之后，
进入ffmpeg目录


.configure/ —enable-gpl —enable-version3 —enable-nonfree —enable-libfaac —enable-libmp3lame —enable-libx264 —enable-libxvid —enable-shared —enable-libopencore-amrnb —enable-libopencore-amrwb —enable-libvorbis —enable-pthreads —disable-ffserver —disable-ffplay —enable-libvpx
make &&make install

出现如下错误：


/usr/bin/ld: /usr/local/lib/libvpx.a(vpx_codec.c.o): relocation R_X86_64_32 against .rodata.str1.1’ can not be used when making a shared object; recompile with -fPIC
/usr/local/lib/libvpx.a: could not read symbols: Bad value
collect2: ld returned 1 exit status
make: * [libavcodec/libavcodec.so.54] Error 1

进入libvpx目录，重新编译，
configure添加 —enable-shared


./configure  —enable-shared
make &&make install

接着进入ffmpeg目录重新编译安装
ok！

顺便了解了下video 的api。

如何禁止video右键，防止视频被人下载。
代码如下：

$(".videoPlayer").on('contextmenu', function(){ return false;});
但是一旦视频全频时，右键又有效了。 怎么解决呢？ 在视频外层包一层div，禁止video的自带的控制栏，自己重写。 自定义定义全屏按钮，全屏时实际上不要将video，而将其外层的div全屏就行了。 全屏代码： 
var docElm = $(".videoArea")[0];    
    if (docElm.requestFullscreen) {
        docElm.requestFullscreen();
    }
    else if (docElm.mozRequestFullScreen) {
        docElm.mozRequestFullScreen();
    }
    else if (docElm.webkitRequestFullScreen) {
        docElm.webkitRequestFullScreen();
    }

日本的网页设计为什么那么老土

其实早就这么觉得了。只是以前对设计不是很关心就没管他。今天，看到这篇文章（日文）：
http://memo.goodpatch.co/2013/02/is-japanese-web-design-bad/
大为赞同。
原本是来自quora的某个提问（英文）。提问者显然后来更改了标题，将so bad改成了so different：
http://www.quora.com/Questions-That-Contain-Assumptions/Why-is-the-design-of-many-Japanese-websites-so-different-from-the-US

当然设计比较不错的也还是有的，不过大多是用的flash。
自己去年来这个学校上体验课，当时的体验课老师（现在教我视觉交流的佐藤老师），给我推荐几个他认为设计不错的网站。
结果也全是flash的。让人觉得有点失望。
另外，大量使用图片，内容多而杂，让人找不到方向。也是日本网站的通病。似乎不知simple为何物。
一个简单例子：日本网站注册时填的表单，那个麻烦啊，让人真想中途放弃 。
相比较而言，日本的工业设计似乎做得相当不错，应该是由于制造业的发达吧。
至于网页方面，或许如上面文章所说，还未成熟。
